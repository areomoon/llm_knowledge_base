## What actually happened on the V5

- Warm-up normal. V4 flashes normal — no anomalous pump, no bicep flag.
- Moved to V5 block. User described it as "做了一個 V5 動作玩" — an
  exploratory single move, not a project attempt.
- Within the attempt (or immediately after), **both biceps** reported
  "超酸痛" — acute soreness, bilateral, simultaneous.
- This is **different** in character from DOMS (delayed) and different
  from pump (localised, transient, forearm-dominant). Bilateral simultaneous
  onset on a single hard pull from a partially-recovered chain matches an
  **acute overload** signal — the working muscle exceeded its current
  capacity ceiling, not its endurance ceiling.

## Observed failure modes / rate-limiters

1. **Bicep / brachialis chain is no longer *only* a dosage question — it is
   a per-move capacity question.** Prior reading (from 04-17 Fri gym
   debrief) was: "exercise selection correct, dosage is the knob". That
   framing assumed the chain fails when accumulated load exceeds capacity.
   Today's data invalidates the pure-dosage frame: **a single hard move,
   early in a well-warmed session, on a partially-recovered chain, was
   enough to trip acute overload.** This is capacity, not dosage. See
   [Climbing Compression Pulling](../../wiki/concepts/climbing-compression-pulling.md)
   for why bicep/brachialis is the *individual-move* limiter on
   deep-reach / compression / lock-off pulls, independent of session
   duration.

2. **Three independent sessions now flag the same chain.**
   - 2026-04-15 Moonboard: `LAST CALL` failed at compression section →
     bicep/brachialis fatigue hypothesised.
   - 2026-04-17 Fri gym: typewriter set 1 near-limit → confirms B1–B3
     already drained the shared pool.
   - 2026-04-21 today: bilateral acute soreness on a single V5 move →
     capacity, not dosage, is now the gate.

   Three independent data points = this is not noise. The bicep chain is
   the project-level rate-limiter for the remaining 4 weeks.

3. **V5-in-flash-bucket was the wrong prescription, even though written
   into the plan.** The 04-21 plan had V5 as "flash-to-2nd-go, max 2 go,
   vertical / ≤ 15°, avoid compression / deep-lock / deep-reach". The V5
   the user attempted either had one of the excluded move types (the most
   likely scenario — V5 at this gym routinely demands a deep reach even
   on nominally-vertical walls), **or** the exclusion list itself is
   insufficient when the bicep chain is in its current state. Either way,
   **V5 volume work should be deferred** until the chain has a clean
   cross-session pattern (≥ 2 consecutive sessions with no bicep flag).

4. **Stop-loss rule worked as designed.** The 04-21 plan's §C "若連續 2
   條路線有明顯牽扯感 → 停 4×4" saved the user from compounding the injury
   into the 4×4 (which would have been 16 continuous ascents on the same
   rate-limited chain under incomplete rest). User actually beat the rule
   — stopped on the first signal rather than the second. Record this as
   a validation: **acute bilateral simultaneous onset should always
   trigger immediate stop, not wait for confirmation.**

## Emergency protocol review (captured mid-session)

The user asked Claude mid-session for acute guidance. The response given:

1. **Hard stop climbing** — no attempting any grade, including V0 straight-
   arm "recovery" work. Bilateral acute = do not re-challenge today.
2. **Skip 4×4** — compounding acute overload with 16 continuous ascents on
   an incomplete-rest protocol is the exact pattern that converts acute
   overload into a strain.
3. **Do not force-stretch the sore biceps** — acute overload ≠ tight
   muscle; aggressive stretching risks the long-head tendon / muscle
   belly. Gentle wrist rotation and light arm shake only.
4. **Ice OR heat acceptable, not mandatory** — ice 10-15 min to blunt
   inflammatory signal, or warm shower for perfusion. User preference.
5. **Conditional Thursday hangboard** — Wed AM elbow-flexor check
   determines whether Thu 04-23 hangboard baseline proceeds. If **any**
   pressure pain on inner elbow or palpable biceps tenderness on Wed AM,
   Thu hangboard is cancelled or reduced to block pulls only (no max
   hangs). Hangboard itself is half-crimp / open-hand (finger-dominant),
   but hanging bodyweight still passes load through a compromised
   elbow-flexor chain — so the check is non-negotiable.

This protocol followed the 04-21 plan's own D-block conditional logic
structure. **Record it as the generalised acute-overload protocol** for
future sessions — extract to KB if the pattern recurs.

## Prescription carried forward (for Wed 4/22 onwards)

1. **Wed 4/22 = acute-recovery day, not active-recovery day.** The
   distinction matters: active recovery (like 04-20 Mon) assumes DOMS
   clearing from an expected post-session response. Acute recovery assumes
   a mid-session overload event that may or may not have caused micro-
   damage. Content: no climbing of any grade, no upper-body loading at
   all, only lower-body blood flow + gentle mobility. Elbow-flexor palpation
   check is the headline KPI.

2. **Thu 4/23 Hangboard baseline is now conditional.** Gate criteria:
   Wed AM biceps palpation tenderness ≤ 2/10 AND elbow inner-side
   pressure pain = 0 AND no sharp twinge on bodyweight-only dead hang
   ≤ 5s. Any fail → Thu = second recovery day, hangboard pushed to Fri
   or skipped for Week 2 entirely.

3. **Fri 4/24 gym day content will change** regardless of Thu outcome.
   Lock-off + compression content is now off the table until the bicep
   chain clears two consecutive sessions with no flag. Replace with:
   lower-body + core + antagonist + open-chain shoulder prehab. Pinch
   block may return (separate from B chain) only if Wed/Thu checks are
   clean.

4. **Sat 4/25 Peak Day V7 Laputa attempt is in serious doubt.** Laputa
   is a compression / deep-lock route by user's own diagnosis — it is
   the precise load pattern the chain cannot currently tolerate.
   Proposal for Wed/Thu re-plan window: downgrade Sat to V5-V6 volume
   day with no project attempts; Laputa pushed to Week 3 Sat 5/2 at
   earliest. User to make the call after seeing Wed/Thu data.

5. **V5-as-flash bucket retired for Week 2.** Revised flash bucket:
   V3 (warm-up), V4 (flash attempts max 2 go). V5 is project-only, which
   in Week 2 means Peak-day only, which — per §4 — means not this week.

6. **Cross-session bicep-chain clean window must be declared before
   re-introducing any V5+ work.** Operational definition: **2 consecutive
   sessions with no bilateral bicep tug, no typewriter-style near-limit
   on set 1, no compression / deep-lock failure** — only then does V5
   flash / Peak V6-V7 return to the program.
